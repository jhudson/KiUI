Date New!

Try setting a date that simply is not a date (for example random text)
Try setting a date that doesn't have the custom format MM-dd-yyyy
This date uses standard italian format dd/MM/yyyy, try setting something in a different format

Personally, I've had some problems using Kendo UI Web date validator: it only works on input[type="date"] and since the DatePicker widget transforms it's <input> element to input[type="text"], you can't use the validator anymore.

Kiui Date validator can be applied to any <input> element and it just works with DatePickers. Additionally, you can also specify a custom culture that the validator will use when trying to parse the date.

To start using the Date validator just add the attribute data-kiui-date to an <input> element and you're done; the date will be validated using every standard format defined for the current default culture.

You can specify a custom format to be used when parsing the date; you can do so by setting the attribute value to the desired format, such as data-kiui-date="dd-MM-yyyy". Now only that format will be used to parse the date.

You can also specify a custom culture to be used when parsing the date instead of the current default one. You can do so by setting the attribute data-kiui-date-culture to the desired custom culture. Now only the standard formats defined for the custom culture will be used to parse the date.

Unfortunately there seems to be a bug in kendo.parseDate() and so the custom culture that you wish to use must have been set at least once as the default culture for the parsing to work correctly. See the Javascript example code for a way to do this.

If you specify both a custom format and a custom culture, only the custom format will be considered when parsing the date.

<div id="validators_dateValue_demo">
    
  <div class="paragraph">
    <form>
      <div class="control-group">
        <label class="control-label">Date validator</label>
        <div class="controls">
          <span class="help-block">Try setting a date that simply is not a date (for example random text)</span>
          <input name="date" data-kiui-date>
          <span class="help-inline"><span class="k-invalid-msg" data-for="date"></span></span>
        </div>
      </div>
      
      <div class="control-group">
        <label class="control-label">Date validator with specific format</label>
        <div class="controls">
          <span class="help-block">Try setting a date that doesn't have the custom format <code>MM-dd-yyyy</code></span>
          <input name="date-format" data-kiui-date="MM-dd-yyyy">
          <span class="help-inline"><span class="k-invalid-msg" data-for="date-format"></span></span>
        </div>
      </div>
      
      <div class="control-group">
        <label class="control-label">Date validator with specific culture</label>
        <div class="controls">
          <span class="help-block">
            This date uses standard italian format <code>dd/MM/yyyy</code>,
            try setting something in a different format
          </span>
          <input name="date-culture" data-kiui-date data-kiui-date-culture="it-IT">
          <span class="help-inline"><span class="k-invalid-msg" data-for="date-culture"></span></span>
        </div>
      </div>
      
    </form>
  </div>
    
</div>


jQuery(function($) {
  
  // set Italian culture
  kendo.culture("it-IT");
  // reset default US culture
  kendo.culture("en-US");
  // I need to do this because there seems to be a bug in kendo.parseDate(): it
  // doesn't work for cultures that have never been set as default culture at
  // least one time
  
  var today = new Date();
  
  $('#validators_dateValue_demo [name="date"]').kendoDatePicker({
    value: today
  });
  
  $('#validators_dateValue_demo [name="date-format"]').kendoDatePicker({
    format: 'MM-dd-yyyy',
    value: today
  });
  
  $('#validators_dateValue_demo [name="date-culture"]').kendoDatePicker({
    culture: 'it-IT',
    value: today
  });
  
  var validator = $("#validators_dateValue_demo form").kendoValidator().data("kendoValidator");
  
});
