Introduction

KiUI is a companion framework for Telerik's Kendo UI Web. It adds widgets, validators, bindings and other higher-level features useful for your everyday job with Kendo... so that you don't have to develop them yourself!! :)

KiUI comes to you from the experience of a professional that works day after day with Kendo framework to build customized widgets and features for web applications and sites.

KiUI is currently tested on the latest version of Chrome and Firefox using Kendo UI Web v2013.1.319.

You had an idea for a new feature that you think would be a perfect addition to KiUI? Just open a new issue or submit a new pull request and tell me all about it!! Together we can try to add it!! :)

Installation

To start working with KiUI begin downloading the latest version, the package contains .css and .js files both in minified and un-minified version.

After you included the Kendo UI Web library and one of its color themes, you must include the KiUI library using the following code: 

<link href="kiui-0.0.2.min.css" rel="stylesheet" />
<script src="kiui-0.0.2.min.js"></script>

Now you're good to go!! Explore all the available KiUI features described below and... happy coding!! :)

Bool Value

Current view-model state:
{
    likeKiui: 
}

The boolValue binding can be used to bind a radio buttons control with a boolean variable in the view model.

It's able to support three different values: true, false and null.

  • Html
  • Javascript
<div id="bindings_boolValue_demo">
    <div class="span7">
        <form>
            <label>Do you like <strong>KiUI</strong>?</label>

            <label class="radio">
                <input type="radio" name="boolValue_demo" value="true" data-bind="boolValue: likeKiui" />
                Yes <span class="help-inline">(Be sure to check this one!! ;-)</span>
            </label>

            <label class="radio">
                <input type="radio" name="boolValue_demo" value="false" data-bind="boolValue: likeKiui" />
                No
            </label>

            <label class="radio">
                <input type="radio" name="boolValue_demo" value="null" data-bind="boolValue: likeKiui" />
                Don't know
            </label>
        </form>
    </div>

    <div class="span5">
        <h5>Current view-model state:</h5>
        <pre>
{
    likeKiui: <span data-bind="text: likeKiuiText()"></span>
}</pre>
    </div>

</div>
jQuery(function($) {
  var viewModel;

  viewModel = kendo.observable({
    likeKiui: null,
    likeKiuiText: function() {
      return kendo.stringify(this.get('likeKiui'));
    }
  });

  kendo.bind($('#bindings_boolValue_demo'), viewModel);

});

The Notifier utility helps you to send notifications to the user.

Start creating an instance of Notifier using the kiui.notifier() function, it accepts a single position option that can have four values: top-left, bottom-left, bottom-right or top-right. Default is top-right.

The Notifier exposes three methods that you can use to create different kinds of notifications: error(), info() and success().

Each method accepts the following options:

  • append: when set to true, the given content will be appended to a previous notification of the same type; if such a notification does not exist, a standard notification will be created instead. (Default: false)
  • autoHide: when set to a number, the notification will be automatically hidden after the given amount of milliseconds. If set to false the notification will have to be hidden manually. (Default: false)
  • content: the html content that will be inserted inside the notification.
  • icon: the html that will be used as icon for the notification, if set to "" the icon will be hidden. (Default: every method has its specific default icon)
  • title: the notification title, if set to "" the title will be hidden. (Default: "")
  • width: the notification width in pixels. (Default: 250)

      The Popup Menu widget allows you to create a menu attached to a specific target element (such as a button or an anchor).

      The basic HTML layout needed to build a Popup Menu requires a container element (usually a <div> tag) that contains exactly two children: the first one is the trigger element which will trigger the menu opening and closing, the second one is a <ul> element that will render the actual menu.

      Under the hood, Popup Menu uses the kendoMenu widget to build the menu so the possible structure for the <ul> subtree is the same as the one described for the kendoMenu widget.

      The trigger element can be any HTML tag but usually a <button> or <a> tag will be used.

      The widget accepts the following configuration options:

      • openOnHover: by default the menu will open when clicking on the target menu; setting this option to true will trigger the menu opening when the mouse stops over the trigger. (Default: false)
      • direction: controls the menu and submenus opening directions; accepted values are 'bottom right', 'bottom left', 'top right' and 'top left'; the first value is the menu direction and the second one is the submenus direction. (Default: 'bottom right')
      • animation: specifies which animation will be used when opening and closing the menu and submenus; available animations are 'expand', 'fade' and 'none'. (Default: 'expand')
      • dataSource: with this option it's possible to build the menu using a dataSource instead of static HTML; for the structure of the dataSource, refer to the kendoMenu widget documentation.

      The widget fires the following events to which you can subscribe:

      • open: fired before the menu or any of the submenus are opened; canceling the event will prevent the opening; the event object will have a property item containing the DOM element representing the opened menu item.
      • close: fired before the menu or any of the submenus are closed; canceling the event will prevent the closing; the event object will have a property item containing the DOM element representing the closed menu item.
      • select: fired when a menu or submenus item is selected; the event object will have a property item containing the DOM element representing the selected menu item.
      You can subscribe to the events as you would do with any other Kendo UI widget: passing the event handler as a configuration property or using the bind() method on the widget object.

  $('#widgets_popupMenu_demo .top-right-direction-option').kendoKiuiPopupMenu({
    direction: 'top right'
  });
  
  $('#widgets_popupMenu_demo .top-left-direction-option').kendoKiuiPopupMenu({
    direction: 'top left'
  });
  
  //animation
  $('#widgets_popupMenu_demo .expand-animation-option').kendoKiuiPopupMenu({
    animation: 'expand'
  });
  
  $('#widgets_popupMenu_demo .fade-animation-option').kendoKiuiPopupMenu({
    animation: 'fade'
  });
  
  $('#widgets_popupMenu_demo .none-animation-option').kendoKiuiPopupMenu({
    animation: 'none'
  });
  
  // dataSource
  $('#widgets_popupMenu_demo .datasource-option').kendoKiuiPopupMenu({
    dataSource: [
      {
        text: "Menu 1",
        spriteCssClass: "k-icon k-i-clock",
        items: [
          {
            text: "<b>SubMenu 1</b>",
            encoded: false
          },
          {
            text: "SubMenu 2"
          }
        ]
      },
      {
        text: "Menu 2",
        spriteCssClass: "k-icon k-i-search"
      }
    ]
  });
  
  // event handling
  var eventLogger = $('#widgets_popupMenu_demo .kiuiEventLogger ul');
  
  $('#widgets_popupMenu_demo .trigger-events').kendoKiuiPopupMenu({
    open: function(e) {
      var itemName = $(e.item).is('button') ?
        'Button menu' : 
        $(e.item).children('.k-link').text().trim();
      
      eventLogger.logEvent("Opened: " + itemName);
    },
    close: function(e) {
      var itemName = $(e.item).is('button') ?
        'Button menu' : 
        $(e.item).children('.k-link').text().trim();
      
      eventLogger.logEvent("Closed: " + itemName);
    },
    select: function(e) {
      eventLogger.logEvent('Selected: ' + $(e.item).children('.k-link').text().trim());
    }
  });
  
  var blockEvent = false;
  
  $('#widgets_popupMenu_demo .cancel-event').kendoKiuiPopupMenu({
    open: function(e) {
      var itemName = $(e.item).is('button') ?
        'Button menu' : 
        $(e.item).children('.k-link').text().trim();
      
      if (itemName === 'Button menu' && blockEvent) {
        e.preventDefault();
       
        eventLogger.logEvent("Open prevented: " + itemName);
      }
      else {
        eventLogger.logEvent("Opened: " + itemName);
      }
      
      if (itemName === 'Button menu') {
         blockEvent = !blockEvent;
      }
    }
  });
  
  // you can also subscribe to the event after the widget was created
  $('#widgets_popupMenu_demo .bind-event').kendoKiuiPopupMenu({});
  // retrieve the widget object
  $('#widgets_popupMenu_demo .bind-event').data('kendoKiuiPopupMenu')

    .bind('select', function(e) {
      eventLogger.logEvent('Selected: ' + $(e.item).children('.k-link').text().trim());
