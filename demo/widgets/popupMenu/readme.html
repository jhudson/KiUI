The Popup Menu widget allows you to create a menu attached to a specific target element (such as a button or an anchor).

The basic HTML layout needed to build a Popup Menu requires a container element (usually a <div> tag) that contains exactly two children: the first one is the trigger element which will trigger the menu opening and closing, the second one is a <ul> element that will render the actual menu.

Under the hood, Popup Menu uses the kendoMenu widget to build the menu so the possible structure for the <ul> subtree is the same as the one described for the kendoMenu widget.

The trigger element can be any HTML tag but usually a <button> or <a> tag will be used.

The widget accepts the following configuration options:

  • openOnHover: by default the menu will open when clicking on the target menu; setting this option to true will trigger the menu opening when the mouse stops over the trigger. (Default: false)
  • direction: controls the menu and submenus opening directions; accepted values are 'bottom right', 'bottom left', 'top right' and 'top left'; the first value is the menu direction and the second one is the submenus direction. (Default: 'bottom right')
  • animation: specifies which animation will be used when opening and closing the menu and submenus; available animations are 'expand', 'fade' and 'none'. (Default: 'expand')
  • dataSource: setting this option it's possible to build the menu using a dataSource instead of static HTML; for the structure of the dataSource, refer to kendoMenu widget documentation.
