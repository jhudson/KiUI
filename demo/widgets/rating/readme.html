Rating widget can be used to rate any kind of content with stars (or other icons that you might like :)

It is highly customizable and very well integrated with Kendo UI Web framework MVVM pattern.

The basic HTML needed to create the widget is a <ul> element containing one <li> element for every possible rating. Every <li> element must have at least a data-value attribute set to the rating value to which the element corresponds.

Configuration options:

  • starEmptyClass: it is the CSS class added to the elements that represent empty stars, a custom class can be used to personalize the appearance of empty stars. (Default: "kiui-rating-star-empty")
  • starFullClass: it is the CSS class added to the elements that represent full stars, a custom class can be used to personalize the appearance of full stars. (Default: "kiui-rating-star-full")
  • value: the initial rating value that is displayed when the widget is created. (Default: null)
  • readonly: if set to true the widget will be initially rendered in readonly mode; its value will not be modifiable by the user but only programmatically. (Default: false)

Methods:

  • value(): used to get/set the current rating value. Without parameters it returns the current rating value, with a single parameter it sets it as the current rating value.
  • readonly(): used to toggle readonly mode. With boolean parameter true or without any parameter the widget will be readonly; with false the rating value will be editable by the user. When the widget is in readonly mode, the rating value can still be modified programmatically (via the value() method or modifying the View Model when using MVVM data binding).

Events:

  • mouseover: fired when the user hovers over a star; the event object will have a property item containing the DOM element representing the star and a property value containing the rating value associated with the star.
  • mouseleave: fired when the user moves the mouse out of the widget.
  • select: fired when the user clicks on a star; the event object will have a property item containing the DOM element representing the star and a property value containing the rating value associated with the star.
  • change: fired after the current rating value has been changed by the user (mainly this is used by MVVM to update the View Model).
